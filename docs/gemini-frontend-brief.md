# 给 Gemini 的项目说明 —— 请你做前端优化建议

你好 Gemini。下面是一个已经在运行的桌面应用项目。我需要你**只针对前端（视觉 / 交互 / 信息层级 / 一致性 / 动效 / 可用性）给出可执行的优化建议**。一个工程师助手（Claude）会拿你的建议直接改代码，所以请尽量具体到"改哪个组件、怎么改"。

---

## 1. 这是什么项目（一句话）

一个**只读**的多账号 Gmail 查看器，Mac/Windows 桌面应用（Electron）。服务一位 Noon 电商卖家，他长期持有 **30+ 个会每隔几天就轮换**的 Gmail 账号，需要在一个窗口里集中查看所有账号的收件箱。

## 2. 用户画像（影响审美与交互取向）

- **非开发者**，中文母语，电商卖家
- macOS 自用，朋友用 Windows
- 核心诉求：**只读**、多账号、少反复登录、不要触发 Google 封控
- 审美取向：**极简、克制、像原生 macOS 应用**（已做过一轮 macOS Sequoia 浅色风格改造）

## 3. 极其重要的硬约束（请不要违背，否则建议会被直接否决）

- **这是个"极简只读查看器"，价值就在极简。不要建议增加任何新功能。** 以下功能用户已**明确拒绝**，请不要提：发送邮件、跨账号搜索、按发件人/主题过滤、附件查看、已读未读标记、长历史邮件（只拉最近 20 封）、系统弹窗通知、统一收件箱。
- **不要碰后端 / 数据层 / 安全模型**：IMAP 连接、Keychain 存储、应用专用密码、TOTP 计算、翻译调用都不在范围内。只优化"看得见的前端"。
- **不要建议引入重型 UI 框架或组件库**（如 MUI / AntD / Chakra）。现有是 Tailwind + 手写组件 + 一套 CSS 变量主题，要保持轻量。
- **不要建议加动画过度的花哨效果**。可以有克制的微交互/过渡，但产品基调是安静、专业、不抢眼。
- 不要建议改成深色模式优先——用户群偏好浅色（已有 3 套浅色主题）。

## 4. 技术栈（前端相关）

- **Electron + React 18 + TypeScript**
- **Tailwind CSS**，颜色全部走 **CSS 变量主题**（`--c-bg / --c-surface / --c-sidebar / --c-border / --c-text / --c-muted / --c-accent` 等），所以做视觉建议时请用"语义色"思路，别写死十六进制。
  - 已有 3 套浅色主题：`cream`（暖米白·靛蓝）、`slate`（冷青灰·系统蓝）、`onyx`（中性深灰·黑），通过 `:root[data-theme="..."]` 切换，有 200ms 全局色彩过渡。
  - 语义色跨主题统一：`danger #ff3b30 / success #34c759 / warning #ff9500`。
  - 字体：`-apple-system` 系列；字号偏小紧凑（base 13px，xs 11px）。
- **zustand** 做状态管理，**lucide-react** 做图标。
- 邮件正文用 **sandboxed `<iframe srcDoc>`** 渲染（CSP 禁 JS、禁外部资源，默认不加载外部图片防追踪像素）。

## 5. 前端结构（你要优化的对象）

顶层：`App` → `MainLayout`（顶部 `Toolbar` + 下面三栏） + 常驻的 `AddAccountDialog`。

**经典三栏布局**（`flex`，整窗 `h-screen`）：

- **顶部 Toolbar**：Logo、刷新按钮、主题切换（Palette 按钮 + ThemePicker popover）、添加账号入口等。
- **左栏 `LeftColumn`**：账号列表（`AccountItem`）。每个账号显示头像（`Avatar`，基于邮箱生成）、邮箱、新邮件未读徽章、状态徽章（🟢正常 / ⚠️应用密码失效 / 🔴同步出错 / 🔄同步中）。底部"+ 添加账号"按钮。
- **中栏 `MiddleColumn`**：当前账号最近 20 封邮件列表（`MessageRow`），顶部有 🔑 凭据抽屉（`CredentialsDrawer`）入口。
- **右栏 `RightColumn`**：选中邮件的详情（`MessageBody` iframe）+ 一键英译中的翻译面板（`TranslationPanel`）。

**添加账号对话框 `AddAccountDialog`**（模态，左右双栏，660×1100）：

- 左侧：表单（Gmail 地址、16 位应用专用密码、TOTP 面板、"粘贴一行账号信息自动解析"、折叠帮助、成功提示横条）。
- 右侧：内嵌 `<webview>` 加载 Google 应用密码页，头部工具栏有"→应用密码页 / →2FA设置 / 退出并清空登录（两步确认）/ 刷新 / 代理设置 / 折叠 / 系统浏览器打开"等按钮。

## 6. 当前已知的前端现状/风格

- 已做过 macOS Sequoia 浅色 UI 改造 + 多主题切换，整体干净。
- 紧凑信息密度（要在一屏里管理 30+ 账号，所以偏紧凑）。
- 圆角、细边框、`backdrop-blur`（Toolbar）、轻投影的 macOS 观感。
