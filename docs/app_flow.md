# 应用流程（App Flow）

> 从用户视角描述"打开 App 会发生什么"，每个按钮点下去的流程长什么样。
> 本文档**不**讲代码实现，只讲用户看到和感受到的东西。

---

## 总览

```
[首次启动] ──无账号──►【空状态 + 添加按钮】
                           │
                           │ 点"+ 添加账号"
                           ▼
                   【添加账号对话框】（左表单 + 右内嵌 Google 应用密码页）
                           │
                           │ 登录 → 生成 → 粘贴 → 验证并添加
                           ▼
  ┌──────────────【主界面：三栏 + 顶部工具栏】──────────────┐
  │  左栏                 中栏               右栏         │
  │  账号列表+头像        邮件列表+头像      邮件正文      │
  │  ⋯菜单（更新/移除）   点邮件→正文        翻译/图片开关 │
  └────────────────────────────────────────────────────┘
```

---

## Flow 1：首次启动（无账号）

**触发**：用户第一次双击打开 App，本地数据库空。

**界面**：
1. 主窗口 3 栏布局
2. 左栏显示 **Logo + "还没有账号 / 点下方按钮添加第一个" + 提示徽章**
3. 中栏显示 **Inbox 图标 + "选择左侧账号查看邮件"**
4. 右栏显示 **MailOpen 图标 + "选择一封邮件查看内容"**
5. 左栏底部蓝色渐变按钮 **"+ 添加账号"**

---

## Flow 2：添加账号（核心流程）

**触发**：点"+ 添加账号"按钮。

**界面**：弹出 `AddAccountDialog`，1100 × 660 的大对话框，左右两栏。

### 左栏（420px）：表单

固定三段式：
- **Header**：标题 "添加 Gmail 账号" + 关闭按钮
- **Scroll**（可滚动）：
  - 成功提示横条（新加完才显示）
  - 📋 **粘贴一行账号信息自动解析** 按钮（点开变成 textarea）
  - 登录辅助区（解析成功后显示 Google 登录密码、辅邮、链接，带复制按钮）
  - ❓ **应用密码怎么生成？**（折叠帮助）
  - **Gmail 地址** 输入框
  - **应用专用密码** 输入框
  - **2FA 验证码生成器** 面板
  - 错误提示（如有）
- **Footer**（固定）：取消/完成 按钮 + **验证并添加** 按钮

### 右栏（剩余宽度）：Google 应用密码页 webview

Header 工具栏：
- **→ 应用密码页**（直接跳转 `myaccount.google.com/apppasswords`）
- 登出 / 刷新 / 折叠 / 系统浏览器打开

内容：Electron `<webview>` 加载 `https://myaccount.google.com/apppasswords`，用户可以在里面登录、操作、复制。

### 操作流程

**场景 A：用户手动填**

1. 在右侧 webview 登录 Gmail → 生成 16 位应用密码 → 复制
2. 左侧填 Gmail 地址 + 粘贴应用密码
3. 点"验证并添加" → App 用 IMAP 验证 → 成功入库 → 表单清空 + 右侧自动登出 → 可以继续加下一个

**场景 B：粘贴一行信息解析**

1. 点左侧"📋 粘贴一行账号信息自动解析"
2. 展开 textarea，粘贴：
   - 格式 1：`账号 密码 辅邮 2fa密钥`（空格分隔，2FA 中的空格不影响）
   - 格式 2：`账号----密码----辅邮----2fa密钥----链接`
3. 点"解析并填入"：
   - Gmail → 填入邮箱输入框
   - 2FA 密钥 → 填入 TOTP 面板（6 位码立即跳动）
   - Google 登录密码 + 辅邮 → 显示在"登录辅助"区
4. 在右侧 webview 登录时：
   - 复制"登录密码"粘贴到 Google 登录页
   - 复制 TOTP 面板的 6 位码粘贴到 2FA 验证页
5. 登录完跳到应用密码页 → 生成 → 复制 → 粘贴到左侧"应用专用密码" → 验证并添加

## Flow 3：查看邮件

**前提**：已添加至少一个账号。

### 左栏：账号列表

每行 44px：
- 圆形头像（邮箱首字母 + 稳定配色）
- 状态徽章（右下角）：🟢 正常 / ⚠️ 应用密码失效 / 🔴 同步出错 / 🔄 同步中
- 用户名（邮箱 @ 前）+ 域名
- hover 时右侧出现 "⋯" 菜单按钮

选中的账号：**白底 + 细阴影 + 淡边框**（Mac 卡片感）。

### 中栏：邮件列表（点账号后显示）

每行：
- 圆形头像（发件人首字母 + 稳定配色）
- 发件人名（加粗）+ 日期
- 主题
- 摘要（已去除 `[image: X]` 和 URL 杂质）

点击某行 → 该邮件变成选中态，右栏显示正文。

### 右栏：邮件正文

- 大号主题
- 发件人卡片：头像 + 姓名 + `<邮箱>` + 时间
- "发给 xxx@gmail.com"
- "**🏞 加载外部图片**" 切换按钮（默认关）
- 翻译面板（下一节单独讲）
- **sandboxed iframe** 渲染 HTML 正文

---

## Flow 4：翻译为中文

**入口**：右栏正文上方的卡片。

### 状态机

- **idle**：显示提示 + "翻译为中文"按钮
- **loading**：转圈 + "翻译中…"
- **done**：显示翻译结果 + 折叠/重新翻译 按钮
- **error**：红色条 + 重试按钮

### 数据流

1. 渲染层取 `bodyText`（首选）或从 `bodyHtml` 剥离标签
2. 通过 IPC 发给主进程
3. 主进程做预处理：
   - 去掉 `[text](url)` markdown 链接残骸
   - 去掉裸 URL、`[image: X]`、装饰线
   - 按 `\n\n` 拆成段落
4. 每段调 `translate.googleapis.com/translate_a/single?client=gtx&sl=auto&tl=zh-CN`
5. 结果按段落用 `\n\n` 拼回
6. 渲染层做"短段落合并"优化（≤40 字的相邻短行合并成一段）

---

## Flow 5：手动刷新

**单账号**：中栏顶部 🔄
**全部**：左栏顶部 🔄

**并发上限 5**，进度通过 IPC 实时推送到 renderer（每个账号的头像徽章变为旋转图标）。

---

## Flow 6：后台自动刷新

**规则**：App 开着，每 1 小时触发一次。

**行为**：
- 按 5 并发分批同步所有账号
- 每个账号 = 一次 IMAP 会话 + 拉最近 20 封 + upsert 本地库
- 遇 AUTHENTICATIONFAILED → 标记为 `expired`，**不重试**
- 通过 IPC `refresh:progress` 推送进度给 UI

**用户感受**：不打扰。下次点过来时，邮件列表已经是新的。

---

## Flow 7：应用密码失效

**触发**：某账号 IMAP 登录失败（AUTHENTICATIONFAILED），被标记为 `expired`。

**UI 表现**：
- 账号头像右下角 ⚠️ 黄色徽章
- hover tooltip："密码失效，点击更新"
- **点击账号 → 不切到邮件视图**，直接弹出"更新应用密码"对话框
- 对话框锁定 email 字段（不能改），只允许输新的应用密码
- 验证成功 → 恢复 `ok` 状态，自动重新同步

---

## Flow 8：移除账号

**触发**：账号行 "⋯" → "移除账号"。

**流程**：
1. **系统原生对话框**二次确认（"确定要移除 xxx 吗？本地缓存会被删除"）
2. 用户确认 →
3. 从 Keychain 删除应用密码
4. 从 SQLite 删除账号行（CASCADE 删除关联的 messages）
5. UI 移除该账号
6. 如果被删的是当前选中的，切到列表第一个账号

**注意**：我们**不会**替用户在 Google 端撤销应用密码。用户自己可以去 `myaccount.google.com/apppasswords` 手动撤销。

---

## Flow 9：App 关闭 / 重启

**关闭**：点红叉，正常退出。

**重启**：
- 读 Keychain 里的所有应用密码
- 读 SQLite 里的账号列表和缓存邮件
- 渲染左栏 + 中栏（上次选中的账号）
- 启动自动刷新定时器（下一次触发在 1 小时后）

---

## 边界行为约定

| 情况 | 行为 |
|---|---|
| 网络离线 | IMAP 连接失败，账号标为 `error`，下次自动重试 |
| IMAP 超时 | 连接 10 秒 / 握手 10 秒 / socket 30 秒超时上限，失败后标错误 |
| 邮件正文超过 5 MB | 目前没限制，iframe 有滚动条能看，性能可能稍差 |
| 收件箱为空 | 中栏显示"暂无邮件" |
| SQLite 文件损坏 | 启动时 better-sqlite3 报错，目前没做自动恢复 |
| 翻译接口不可用 | 翻译面板显示红色错误 + 重试按钮，不影响邮件查看 |
